Korlátozások Lekérdezések

1. Táblákban alkalmazható korlátozások (constraints)

PRIMARY KEY

Azonosítja a tábla egyedi sorait; nem lehet NULL és egyedi értékeket biztosít.

CREATE TABLE Pelda (id INT PRIMARY KEY, nev VARCHAR(100));

FOREIGN KEY

Másik tábla rekordjára hivatkozik, biztosítva az adatintegritást (kapcsolat).

CREATE TABLE Rendeles (id INT PRIMARY KEY, vasarlo_id INT, FOREIGN KEY (vasarlo_id) REFERENCES Vasarlo(id));

UNIQUE

Megköveteli, hogy az oszlopban minden érték egyedi legyen (de NULL lehet, ha nincs NOT NULL).

CREATE TABLE Felhasznalo (id INT PRIMARY KEY, email VARCHAR(255) UNIQUE);

NOT NULL

Megakadályozza, hogy az oszlop értéke NULL legyen — kötelező mező.

CREATE TABLE Termek (id INT PRIMARY KEY, nev VARCHAR(100) NOT NULL);

DEFAULT

Alapértelmezett értéket ad meg, ha az INSERT során nem adnak meg értéket.

CREATE TABLE Allapot (id INT PRIMARY KEY, aktiv BOOLEAN DEFAULT TRUE);

CHECK

Feltét határoz meg az oszlop értékeire (egyszerű validáció az adatbázis szintjén).

CREATE TABLE Munkavallalo (id INT PRIMARY KEY, eletkor INT CHECK (eletkor >= 18));

2. A lekérdezések típusai

Adatbevitel (INSERT)

Új sorokat ad a táblához. Lehet oszlopsorárnyalás vagy teljes sor.

INSERT INTO Vasarlo (id, nev, email) VALUES (1, 'Kovács Anna', 'anna@example.com');

Adatlekérdezés (SELECT)

Adatok lekérése a táblákból; egyszerű vagy összetett feltételekkel, csatlakozásokkal.

SELECT v.nev, r.id AS rendeles_id FROM Vasarlo v JOIN Rendeles r ON v.id = r.vasarlo_id WHERE v.email LIKE '%@example.com';

Adatmódosítás (UPDATE)

Meglévő sorok értékeinek módosítása feltételek alapján.

UPDATE Termek SET ar = ar * 1.1 WHERE kategoria = 'elektronika';

Adattörlés (DELETE)

Sorok eltávolítása a táblából; óvatosan használandó, gyakran tranzakcióban.

DELETE FROM Rendeles WHERE datum < '2020-01-01';
